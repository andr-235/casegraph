# CaseGraph

## Обзор

Десктопное приложение для управления делами (case management) на базе Tauri v2 + React 19 + TypeScript + SQLite. Офлайн-режим, три роли пользователей, визуализация связей, временная шкала, аудит и резервное копирование.

## Основные возможности

- Управление делами: создание, редактирование, статусы, код дела (CASE-001, ...)
- Объекты дела: документы, персоналии, локации, события с произвольными атрибутами
- Материалы: файловое хранилище, типы материалов, проверка целостности
- Связи (relation-based): графовая визуализация связей между объектами, настраиваемые типы и уровни уверенности
- Временная шкала: события с точностью дат, фильтрация, отчёты
- Аудит: полный журнал действий с безопасными снепшотами, экспорт
- Резервное копирование: ZIP-архивы с проверкой и восстановлением
- Разграничение прав: матрица из 27+ операций, ролевые и политико-зависимые доступы

## Технологический стек

- **Язык программирования:** Rust (бэкенд), TypeScript (фронтенд)
- **Фреймворк:** Tauri v2 (десктопная оболочка), React 19
- **База данных:** SQLite (rusqlite, bundled)
- **Сборка:** Vite 7, tsc
- **Аутентификация:** argon2, парольное хэширование, in-memory сессии
- **Безопасность:** трёхуровневая авторизация (ProtectedServiceContext → PolicyAwarePermissionGuard → PolicyAwarePermissionService)

## Архитектура

Подробные архитектурные правила, зависимости слоёв и примеры кода — см. [ARCHITECTURE.md](ARCHITECTURE.md).

**Паттерн:** Structured Modules (Technical Layers)

## Архитектурные решения

- **Feature-based** на фронтенде: `features/{auth,cases,materials,...}/` с поддиректориями `api/`, `model/`, `ui/`
- **Layer-based** на бэкенде: `commands/` → `services/` → `repositories/` → `db/`
- Без внешнего роутера: навигация управляется через enum `BootstrapState` в `App.tsx`
- Единый контракт API: `CommandResult<T>` = `{ ok: true, data: T } | { ok: false, error: AppErrorDto }`
- Обёртка `invokeCommand` на фронтенде: разворачивает `CommandResult`, выбрасывает `AppCommandError`
- Миграции БД выполняются идемпотентно при каждом `initialize_app`

## Нефункциональные требования

- **Офлайн-режим:** приложение полностью автономно, `offlineMode: true`
- **Интерфейс:** русский язык
- **Обработка ошибок:** структурированные коды ошибок (`ERR_DATABASE`, `ERR_ACCESS_DENIED`, `ERR_VALIDATION`, ...)
- **Безопасность:** безопасные снепшоты аудита (санитизация путей), best-effort запись без прерывания основного потока
